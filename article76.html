Git Flow 实践

一，Git 分支模型


二，SourceTree 中项目加入 Git Flow

1，仓库初始化：点 SourceTree 中右上角的 Git Flow（注意第二个选项中：Development branch：develop 改成实际的 dev）-> OK

2，再点右上角的 Git Flow -> Start a New Feature -> 填写 Feature 名称（Feature 只与 dev 分支交互）

3，Feature 完成后，先把 dev 合并到当前 Feature 分支（如果有冲突，要先解决冲突），再点右上角的 Git Flow -> 完成功能 -> 此时会合并到 dev（推荐勾选：删除当前 Feature 分支）


三，Git Flow 在 完成功能的时候出现错误：fatal: could not read Username for 'https://gitee.com': No such file or directory

1，进入命令行模式，执行：

git flow feature finish -k 功能名称


四，Git Flow 在 rebase（变基）的时候出现代码冲突的解决方法

1，前提：基于 dev 建立 feature 分支后，dev 又与 feature 修改了同一个文件

2，feature 完成功能前先将 dev 拉取到 feature -> 完成功能 -> 不勾选：删除feature分支
