培训 1 - Git + SourceTree

一，基本操作

1，安装 SourceTree：

参考：http://huanglu.cn/article65

2，安装 Git：

下载地址：https://git-scm.com/downloads

3，下载代码：

GitLab 仓库地址：http://192.168.90.238:8091


二，SourceTree 的基本操作

1，正常提交代码：

第一步：编写完成后提交本地

第二步：拉取服务器上的代码

第三步：一起推送到服务器上

2，储藏代码：

储藏所有代码，保持工作区没有暂存文件

3，解决冲突：

第一步：出现问题，修改自己的问题

第二步：右键冲突的文件 -> 解决冲突 -> 标记为已解决

第三步：提交


三，SourceTree 的分支操作

1，创建分支：

第一步：点分支 -> 创建新分支，会根据仓库当前HEAD 指向的分支来进行创建 -> 填写新分支名称；

第二步：创建新分支后，SourceTree（Git）会默认把HEAD指向新创建的分支，并且新分支与原来的分支所处的状态一样。

第三步：如果要切换分支，只需要在需要切换的分支上双击即可。然后把Dev分支推送到服务端。在sourceTree远程选项的origin中，就会多了一个Dev分支，表明服务端也已经有了Dev分支。到服务端查看是否创建Dev分支成功。好，现在我们可以在Dev分支下工作了。

第四步：第一次创建的分支不需要拉取，可以勾选：立即推送变更到 origin/dev，这样就会在服务器上创建一个新分支。

2，删除分支：

第一步：删除分支时，要保证HEAD不在删除的分支上。比如现在，HEAD指向的就是temp分支，我们在temp分支上

第二步：右键 -> 删除分支 -> 弹出对话框 -> 确认 ->报错。此时，就会报误。就是说要删除的分支有过新的提交，但是还没有进行合并。SourceTree会保证你的删除是可靠的，要求你合并之后再删除。如果，你确定不需要合并，那么在删除分支时选择‘强制删除’。

3，合并分支：

到目标分支上去操作，比如我们要把Dev分支上的修改合并到master分支，那么我们就要到master分支上去操作。选择要合并的节点

4，修改 master 分支的 bug：

第一步：从 master 新建一个临时的 temp分支

第二步：在 temp 分支上修复bug

第三步：合并到 master 分支，这样 master 分支才可以继续发布

第四步：合并到 dev 分支，这样也保证 dev 分支上的代码是稳定的

第五步：删除 temp 分支

5，版本回退：

第一步：选择需要回退的版本 -> 右键 -> 重置当前分支到此次提交 -> 默认选择: 混合合并，然后确定；

第二步：回退的版本并没有丢失，而是在暂存区

第三步：修改正确的代码 -> 解决冲突 -> 提交代码 -> 推送

6，发布新版本：master 合并完之后，就可以发布新版本了。我们使用 tag 标签来表示版本。Tag其实跟分支类似，只不过它是在某一个提交基础上建立的一个指针副本，因此它不能进行移动，但是可以添加多个tag，也可以删除。

第一步：在 master 分支上，点标签，输入有意义的标签名称 -> 提交

第二步：如果在创建tag的时候没有勾选推送，则这里将标签推送到服务端。右键 -> 推送到 -> origin
