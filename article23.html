Jenkins 基础

一，Linux CentOS 6 下安装 Jenkins

1，下载仓库：wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat-stable/jenkins.repo

2，安装仓库：rpm --import http://pkg.jenkins.io/redhat-stable/jenkins.io.key

3，安装：yum -y install jenkins

4，启动服务：systemctl start/stop/restart jenkins.service 或者：/etc/init.d/jenkins start

5，服务自启动：chkconfig jenkins on

6，修改配置：vim /etc/sysconfig/jenkins

三，Mac 下安装 Jenkins

1，brew install jenkins

2，brew services start jenkins 或者直接 jenkins

3，输入：http://localhost:8080，即可访问

4，密码：cat /root/.jenkins/secrets/initialAdminPassword或：cat /var/lib/jenkins/secrets/initialAdminPassword

5，日志位置： tailf /var/log/jenkins/jenkins.log 


四，Jenkins 运行

1，启动一：service jenkins start --httpPort=8081

2，启动二：java -jar /usr/lib/jenkins/jenkins.war --httpPort=8081


五，Jenkins 配置

1，第一次访问要解锁：密码在启动日志中、或：/root/.jenkins/secrets/initialAdminPassword

2，在 Customize Jenkins 中选择：手动安装插件

3，设置：管理员的账户、密码和邮件地址

4，配置：系统管理 -> Global Tool Configuration -> 配置 JDK 和 Maven


六，Jenkins 使用

基础1，关联本地的JDK 和 Maven

基础2，安装插件：Git、Gitlab、Gitlab Hook等

1，创建一个 Maven 风格的项目

2，填写项目名称

3，丢弃旧的构建，3个参数必须都要写，否则无效：选 Log Rotation ；保持构建的天数：30；保持构建的最大个数：30。

4，源码管理 -> 选Git；填写 Repository URL；选择授权；下面的Branches to build 默认是：*/master

5，构建触发器：Build when a change is pushed to GitLab. GitLab webhook URL: http://1.2.3.4:15436/project/project_name

6，Build -> 1，pom.xml；2，clean package -Dmaven.test.skip -e -U

7，Post Steps -> Run regardles of build result

8，配置 SSH：在首页点 Credentials，进去点 Update，Scope 选 Global (…) ，用户名：邮件地址，Private Key 来自 cat ~/.ssh/id_rsa

9，构建触发器打开勾子（hook），GitHub hook trigger for GITScm polling

10，构建设置 -> E-mail Notification，输入email地址

八，Jenkins 问题

1，无法登录的时候，修改系统账户下的 .jenkins/config.xml，将useSecurity 设为false

2，设置登录的配置

3，设置国内镜像：Plugin Manager -> Advanced -> Update Site -> https://mirrors.tuna.tsinghua.edu.cn/jenkins/updates/update-center.json

4，如果说Java 的编辑级别太低，是之前没有删除低版本的Java

5，如果启动的时候显示：Jenkins bash: /usr/bin/java: No such file or directory，建一个软连接：ln -s /home/JDK8/bin/java /usr/bin/java（检查软链接有没有生效： which java）
